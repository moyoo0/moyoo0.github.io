HashMap 键的 equals() 和 hashCode()

当你将一个对象作为 HashMap 的键（Key）时，HashMap 会使用这个键的 hashCode() 和 equals() 方法。如果你使用自定义对象作为 HashMap 的键，你必须为你自己的类重写 equals() 和 hashCode() 方法。

为什么必须重写？

HashMap 的工作机制依赖于这两个方法：

  1. put(key, value) 操作：当你向 HashMap 中 put 一个新的键值对时，HashMap 会首先调用 key 的 hashCode() 方法来计算哈希值，并根据这个哈希值来确定这个键值对应该存储在内部数组的哪个位置（桶）。
  2. 处理哈希冲突：如果不同的键计算出了相同的哈希值（即发生了哈希冲突），HashMap 会在这个位置形成一个链表或红黑树。为了在这个数据结构中找到正确的位置，HashMap 接着会调用键的 equals() 方法来比较它们，以区分不同的键。
