基于spring-ai实现MCP-Server/Client

此文档仅供个人整理学习使用

功能概述

基于 Spring-ai 框架实现如下功能：以 openai 作为 LLM，自定义负责自动发贴的 MCP-server，以 stdio 方式与 LLM 进行交互。

sequenceDiagram
    participant TriggerJob as 定时任务
    participant ChatClient as ChatClient
    participant MCPClient as MCP Client
    participant MCPServer as MCP Server (mcpjar)
    TriggerJob->>ChatClient: prompt(userInput).call()
    ChatClient->>MCPClient: 发现/调用 MCP 工具
    MCPClient->>MCPServer: 通过 STDIO 发送 JSON-RPC 请求
    MCPServer-->>MCPClient: STDIO 返回 JSON-RPC 响应
    MCPClient-->>ChatClient: 返回工具调用结果
    ChatClient-->>TriggerJob: 返回最终内容

MCP Client端实现

1. 引入pom

  • 这是 MCP Client（模型上下文协议客户端）的 Spring Boot 启动器，基于 WebFlux（响应式 SSE）实现。

  • 让你的 Spring Boot 应用可以作为 MCP Client，通过 HTTP SSE 协议/stdio 协议与 MCP Server 通信，实现工具发现、Prompt调用、资源访问等。

  • 自动配置 MCP Client 的所有组件，包括连接管理、协议处理、工具回调集成、SSE传输等。

  • 支持响应式编程模型，适合高并发、异步场景。

3. 构造客户端

OpenAiChatModel openAiChatModel

  • 这是 Spring AI 的“聊天模型”对象，封装了与 OpenAI（或兼容 API，如 Azure OpenAI）通信的底层逻辑。

  • 你可以通过它直接调用 LLM（如 GPT-4、GPT-3.5）。

  • 在本项目中，ChatClient 既可以用 openai，也可以通过 MCP 工具调用外部 AI 服务。

ToolCallbackProvider syncMcpToolCallbackProvider

  • 这是 MCP 客户端的关键参数，类型为 ToolCallbackProvider，实际由 SyncMcpToolCallbackProvider 实现。

  • 它内部持有一组 McpSyncClient，每个 client 代表一个 MCP Server 的连接（如 mcp-server-csdn、mcp-server-weixin）。

  • 作用：将 MCP Server 暴露的“工具”注册到 ChatClient 体系中，这样 prompt 里就可以通过工具调用远程 MCP Server 的 AI 能力。

  • 其构造依赖于配置文件（如 mcp-servers-config-4.json），自动管理 MCP Server 的进程和连接。

ChatMemory chatMemory

  • 对话记忆对象，类型为 ChatMemory，本例用的是 InMemoryChatMemory。

  • 作用：为 ChatClient 提供会话记忆能力，支持上下文多轮对话、历史消息追踪等。

  • 通过 PromptChatMemoryAdvisor 集成到 ChatClient，自动管理 prompt 的上下文。
