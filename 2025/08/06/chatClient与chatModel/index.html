ChatClient与ChatModel

ChatModel

ChatModel：核心原子能力

ChatModel 是与底层 AI 大模型进行交互的 核心原子 API。

  • 职责：它的主要职责是接收一系列消息（ChatMessage）作为输入，然后直接与具体的 LLM 服务（如通义千问、GPT-4 等）进行通信，并返回原始的 ChatResponse。
  • 功能：它提供最基础、最原始的文本交互能力，不包含任何额外的增强功能或复杂逻辑。
  • 类比：你可以把它看作是一个 HTTP 请求库。它知道如何格式化请求并发送到服务器（LLM 服务），以及如何解析响应，但它本身不处理业务逻辑。

1. 输入 (Inputs)

  • Prompt: 用户的核心输入，可以是一个问题、一段文本或指令。
  • Instructions: 指令，它是更广泛的输入，可以包括文本、音频、图像、视频等，具体取决于模型的类型。
  • Chat Options (Start-Up): 在 ChatModel 初始化时设置的全局或默认配置。例如，你可以设置默认使用的 AI 模型（如 GPT-4）或温度（temperature）等参数。这些选项对所有请求都有效，除非被覆盖。
  • Chat Options (Runtime): 每次请求时动态传递的配置。这些选项具有更高的优先级，会覆盖 Start-Up 时设置的任何默认选项。

2. 模型核心处理 (ChatModel / StreamingChatModel)

这是整个流程的核心，它充当了 Spring AI 和底层原生大模型之间的“桥梁”。

  • Convert Input: 这一步将 Spring AI 抽象的输入（Prompt 和 Instructions）转换成底层原生 LLM API 所需的请求格式。例如，将一个 Prompt 对象转换成一个 JSON 格式的 Request。
  • Merge Options: 这一步会合并 Start-Up 和 Runtime 的 Chat Options。如果 Runtime 中设置了某个参数，它会优先使用 Runtime 的值，否则使用 Start-Up 的默认值。
  • 调用原生 LLM API: 处理后的请求被发送到底层 AI 模型。图中的 AI Model 代表了实际的大语言模型服务（如 OpenAI 的 API、通义千问等）。模型处理请求后，会返回一个原生格式的 Response。
  • Convert Output: 这一步将原生模型的 Response 转换回 Spring AI 定义的标准输出格式，也就是 ChatResponse。

3. 输出 (Output)

  • ChatResponse: 这是 ChatModel 返回给你的最终标准响应对象。它包含了 AI 模型生成的所有信息。
  • Generations: 响应的核心部分，代表了 AI 模型的生成结果。
  • Output: 最终生成的内容，通常是纯文本。
  • Metadata: 包含与本次请求和响应相关的额外元数据，比如使用 token 的数量、模型 ID 等。

ChatClient

ChatModel 和 ChatClient 在 Spring AI Alibaba 中扮演着不同的角色，它们之间的区别可以这样理解：

ChatClient：上层业务封装与增强

ChatClient 是在 ChatModel 之上构建的，它是一个 更高级、更易用、功能更丰富 的 API。

  • 职责：它的职责是为开发者提供一个更“流畅”（Fluent API）的接口，并集成各种增强功能，简化与 AI 模型的交互。
  • 功能：ChatClient 不仅能调用 ChatModel 的基础能力，还能通过 Advisor 机制来增强 AI 的行为。例如，它能：
    • 管理对话记忆：通过 PromptChatMemoryAdvisor 让 AI 记住多轮对话的上下文。
    • 集成外部知识：通过 RagAnswerAdvisor 实现检索增强生成（RAG），让 AI 在回答时参考外部知识库。
    • 处理流式响应：提供 stream 方法来处理异步、逐步返回的响应。
  • 类比：如果 ChatModel 是 HTTP 请求库，那么 ChatClient 就是一个 功能齐全的 Web 框架。它在底层 HTTP 请求之上封装了路由、会话管理、数据处理等一系列高级功能，让你能更高效地构建应用。

处理ChatClient响应

call() 返回值

ChatClient.call() 方法支持几种不同类型的响应格式。

  • String content()：返回响应的字符串内容
  • ChatResponse chatResponse()：返回ChatResponse包含多个代以及有关响应的元数据的对象，例如，使用了多少个令牌来创建响应。
  • 返回 Java 类型
    • entity(ParameterizedTypeReference type)：用于返回实体类型的集合。
    • entity(Class type): 用于返回特定的实体类型。
    • entity(StructuredOutputConverter structuredOutputConverter): 用于指定一个实例 StructuredOutputConverter，将 String 转换为实体类型。

stream() 返回值

您还可以调用该stream方法而call不是，在指定stream方法后ChatClient，响应类型有几个选项：

  • Flux<String> content()：返回由AI模型生成的字符串的Flux。
  • Flux<ChatResponse> chatResponse()：返回对象的 Flux ChatResponse，其中包含有关响应的附加元数据。

ChatClient默认设置

  • defaultOptions(ChatOptions chatOptions)：传入 ChatOptions 类中定义的可移植选项或特定于模型实现的如 DashScopeChatOptions 选项。
  • defaultFunction(String name, String description, java.util.function.Function<I, O> function)：name 用于在用户文本中引用该函数，description解释该函数的用途并帮助 AI 模型选择正确的函数以获得准确的响应，参数 function 是模型将在必要时执行的 Java 函数实例。
  • defaultFunctions(String... functionNames)：应用程序上下文中定义的 java.util.Function 的 bean 名称。
  • defaultUser(String text)、defaultUser(Resource text)、defaultUser(Consumer<UserSpec> userSpecConsumer) 这些方法允许您定义用户消息输入，Consumer<UserSpec>允许您使用 lambda 指定用户消息输入和任何默认参数。
  • defaultAdvisors(RequestResponseAdvisor... advisor)：Advisors 允许修改用于创建 Prompt 的数据，QuestionAnswerAdvisor 实现通过在 Prompt 中附加与用户文本相关的上下文信息来实现 Retrieval Augmented Generation 模式。
  • defaultAdvisors(Consumer<AdvisorSpec> advisorSpecConsumer)：此方法允许您定义一个 Consumer 并使用 AdvisorSpec 配置多个 Advisor，Advisor 可以修改用于创建 Prompt 的最终数据，Consumer<AdvisorSpec> 允许您指定 lambda 来添加 Advisor 例如 QuestionAnswerAdvisor。

您可以在运行时使用 ChatClient 提供的不带 default 前缀的相应方法覆盖这些默认值。

  • options(ChatOptions chatOptions)
  • function(String name, String description, java.util.function.Function<I, O> function)
  • functions(String... functionNames)
  • user(String text)、user(Resource text)、user(Consumer<UserSpec> userSpecConsumer)
  • advisors(RequestResponseAdvisor... advisor)
  • advisors(Consumer<AdvisorSpec> advisorSpecConsumer)

Advisors

“advisor”（顾问）是 Spring AI Alibaba 中一个非常重要的概念，它采用了 拦截器链 的设计模式(类似于 Spring 中的AOP)。

advisor 的核心作用和工作方式

  1. 修改传入的 Prompt：在发送请求给 AI 模型之前，advisor 能够截获这个请求并对其进行修改。例如，你可以通过它来注入 检索数据（Retrieval Context） 和 对话历史（Chat Memory）。
  2. 增强模型能力：通过注入额外的数据，advisor 使得 AI 模型的能力得以增强。例如，它让模型能够在同一个会话中记住之前的聊天历史，或者在回答问题时参考外部的知识库（通过 RAG）。
  3. 自定义逻辑：你可以编写自己的 advisor，来实现在请求和响应流程中的任何自定义操作，比如日志记录、数据格式转换等。
